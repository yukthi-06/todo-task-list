# Remove Unnecessary Toast Messages Plan

> remove unnecessary Toast messages

## Proposed Changes

We will remove the success/info Toast messages that are redundant or unnecessary because the action's result is immediately visible in the UI or because the screen closes.
Specifically:
1. "Settings saved to settings.json" in `SettingsActivity.java` and `SettingsFragment.java`.
2. "Master list renamed" in `MainFragment.java`.
3. "Master list created" in `MainFragment.java`.
4. "List moved to 'deleted' folder" in `MainFragment.java`.

Validation error Toast messages (like "Path cannot be empty", "Name cannot be empty", "List not found") will be kept so the user is aware of input errors.
