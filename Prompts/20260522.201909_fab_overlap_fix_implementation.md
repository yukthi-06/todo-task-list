# User Prompt
```
The floating + button is always blocking the last row's delete icon...
Anything can be done about it?
```

# Implementation Plan: Fix FAB Overlapping List Content

We need to prevent the Floating Action Button (FAB) from blocking/overlapping the last row's delete button (and other list content) when scrolling to the bottom of the lists.

To resolve this elegantly and maintain a premium user experience, we will apply the Material Design standard recommendation:
1. **Set `clipToPadding="false"`**: This ensures that list content is not clipped by padding.
2. **Add generous `paddingBottom`**: Set `android:paddingBottom="88dp"` (or similar) on the `RecyclerView` so that when scrolled to the very end, there is extra scrollable space underneath the last item to elevate it completely above the FAB.

We will apply this to the two main lists in the app:
1. **Master List**: `fragment_main.xml`
2. **Todo Items Detail List**: `fragment_list_detail.xml`

## Steps

### Step 1: Modify `fragment_main.xml`
Update the `RecyclerView` (`@id/recyclerView`) to use explicit start, top, and end padding, and set `android:paddingBottom="88dp"` while keeping `android:clipToPadding="false"`.

### Step 2: Modify `fragment_list_detail.xml`
Update the `RecyclerView` (`@id/recyclerViewItems`) to set `android:clipToPadding="false"` and `android:paddingBottom="88dp"`.
